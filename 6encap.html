Encapsulation? What is that?

Encapsulation in Java is a fundamental concept in object-oriented programming (OOP) that refers to the bundling of data and methods that operate on that data within a single unit, which is called a class in Java. Java Encapsulation is a way of hiding the implementation details of a class from outside access and only exposing a public interface that can be used to interact with the class.

In Java, encapsulation is achieved by declaring the instance variables of a class as private, which means they can only be accessed within the class. To allow outside access to the instance variables, public methods called getters and setters are defined, which are used to retrieve and modify the values of the instance variables, respectively.

Here is an example of Encapsulation

public class Area {
private int width
 private int length
Area(int width, int length) {
this.width = width;
 this.length = length;
 }
public void getArea() {
int area = width * length;
 System.out.println("The area is: " + area);
 }
}
public class Main{
public static void main(String[] args) {
 Area rectangle = new Area(8, 6);
 rectangle.getArea();
 }
}

Output:

The area is: 48

Encapsulation allows the internal state of an object to be hidden from external access. By declaring the data members of a class as private, they cannot be accessed directly from outside the class. Instead, access to these members is controlled through methods, enabling the class to maintain control over its data and ensuring its integrity.

Encapsulation enables fine-grained control over access to an object's members. By providing public methods to access and modify the object's state while keeping its data private, encapsulation ensures that the object's integrity is maintained and prevents unintended manipulation or corruption of its internal state.
